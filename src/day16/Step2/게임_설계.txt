Text 와 Console 로 즐기는 메이플

[CRUD 설계]
> C (유저) : 회원 가입, 캐릭터 생성, 장비 착용, 신규 아이템 획득, 아이템 구매, 경매장에 아이템 등록, 퀘스트 받기, 신규 스킬 배우기
> C (GM)  : 신규 직업 생성, 스킬 추가, 신규 장비 등록, 상점에 신규 아이템 등록, 신규 몬스터 추가, 신규 퀘스트 추가, 신규 사냥터 추가

> R (유저) : 내 회원 정보(+ 아이디, 비밀번호 찾기), 내 캐릭터 정보, 내가 갖고 있는 아이템 출력, 내가 받은 퀘스트 출력, 내가 경매장에 올린 아이템 출력
> R (GM)  : 회원 정보(전체), 전체 캐릭터 정보, 전체 아이템(장비, 소비, 기타) 목록 출력, 전체 퀘스트 출력, 몬스터 정보 출력, 경매장 등록 현황

> U (유저) : 회원 정보 수정(비밀번호 변경), 캐릭터 스펙업(경험치, 레벨, 스탯), 원래 있던 아이템 사용 또는 획득(갯수 감소 OR 증가), 스킬 레벨업, 내 캐릭터의 체력 / 마나 , 보스 몬스터의 체력, 현재 퀘스트 진행 상황
> U (GM)  : 캐릭터 밸런스 조정(스킬 데미지), 몬스터 조정(경험치, 체력, 데미지), 보스 몬스터 등장 확률 조정, 사냥터 조정, 퀘스트 수정, 상점 아이템 이름 수정, 장비 능력치 수정

> D (유저) : 회원 탈퇴, 캐릭터 삭제, 장비 해제, 아이템 버리기, 아이템 판매, 퀘스트 완료, 경매장에서 아이템 구매
> D (GM)  : 캐릭터 밸런스 조정(스킬 삭제), 몬스터 삭제, 사냥터 삭제, 퀘스트 삭제, 아이템 삭제, 장비 삭제

[인게임에서의 CRUD 예시]
    > 캐릭터와 상점 npc 간의 거래를 생각해보자.
        > 내 캐릭터의 아이템 테이블과 상점 아이템 테이블 출력 > (코드) "select *from 내 캐릭터의 아이템 테이블","select *from 상점 아이템 테이블"     -> R
        > 그 이전에 GM이 상점 아이템에 아이템 목록(레코드) 추가 > (코드) "insert into 상점 아이템 테이블(필드명) values(추가할 아이템)"  -> C
        > 내 캐릭터가 상점 아이템에서 사고 싶은 아이템을 클릭 > (코드) "select *from 상점 아이템 테이블 where 아이템 번호" = 사고 싶은 아이템 번호 -> R
        > 아이템 구매 완료 시 내 캐릭터의 아이템 테이블에 없는 아이템이라면 아이템 테이블에 추가 > (코드) "insert into 내 캐릭터의 아이템 테이블 values(구매할 아이템)"-> C
        > 아이템 구매 완료 시 내 캐릭터의 아이템 테이블에 이미 있는 아이템이라면 갯수를 증가 > (코드) "update 내 아이템 테이블 set 아이템 갯수의 필드명 = (기존 갯수 + 구매한 갯수) where 내 아이템의 필드명 = 구매한 아이템의 필드명" -> U
        > 아이템 판매 시 내가 팔고 싶은 아이템 클릭 > (코드) "select *from 내 아이템 테이블 where 아이템 번호 = 팔고 싶은 아이템 번호" -> R
        > 판매 완료 시 내가 갖고 있는 아이템 수량을 전부 팔았다면 그 아이템의 목록(레코드)을 삭제 > (코드) "delete from 내 아이템 테이블 where 아이템 식별 번호 필드 = 팔고 싶은 아이템"-> D
        > 판매 완료 시 내가 갖고 있는 아이템 수량을 다 팔지 않았다면 그 아이템의 갯수를 감소 > (코드) "update 내 아이템 테이블 set 아이템 갯수의 필드명 = (기존 갯수 - 판매한 갯수) where 내 아이템의 필드명 = 판매한 아이템의 필드명" -> U

[테이블 설계]
    > 캐릭터 입장의 테이블 : 회원, 내 캐릭터 목록, 내 캐릭터의 스펙, 내 캐릭터가 착용한 장비, 내 캐릭터가 소유한 아이템, 내 캐릭터가 수락한 퀘스트, 내 캐릭터의 스킬, 내 캐릭터의 체력(보스 몬스터 전투에서 사용), 내 캐릭터가 등록한 경매장 아이템
    > GM 입장의 테이블    : 회원, 전체 캐릭터 목록, 전체 직업 목록, 상점 아이템 목록, 전체 사냥터 목록, 전체 보스 몬스터 목록, 전체 퀘스트 목록, 전체 스킬 목록, 보스 몬스터의 체력(보스 몬스터 전투에서 사용), 경매장에 등록한 전체 아이템 목록

[시나리오 구현] (유저)
    > 1. [홈페이지]
        > 1-1. 유저 1이 회원가입을 합니다. 아이디, 비밀번호, 주민번호를 입력받고 아이디 생성
        > 1-2. 유저 1이 로그인을 합니다. if) 로그인 실패 -> 아이디를 찾습니다. -> 주민번호 입력 -> 아이디 찾기 성공 -> 비밀번호를 찾습니다. -> 아이디 입력 -> 비밀번호 찾기 성공
        > 1-3. 유저 1이 회원 정보 수정을 합니다. 주민번호, 아이디 입력 -> (주민번호와 아이디가 맞다면) 기존 비밀번호 입력 -> 새 비밀번호 입력 -> 비밀번호 변경 완료
        > 1-4. 유저 1이 회원 탈퇴를 합니다. 주민번호, 아이디 입력 -> (주민번호와 아이디가 맞다면) 회원 탈퇴를 하시겠습니까? [ Y / N ] -> Y 입력 -> 회원 탈퇴 완료

    > 2. [게임 시작 페이지]
        > 2-1. 유저 1이 신규 캐릭터를 생성합니다. 직업 선택 -> 성별 입력 -> 닉네임 입력 -> 캐릭터 생성 성공
        > 2-2. 유저 1이 생성한 캐릭터 정보를 봅니다. 캐릭터 선택 -> 캐릭터 능력치 출력
        > 2-3. 유저 1이 캐릭터 삭제를 시도합니다. 캐릭터 선택 -> 캐릭터를 삭제하시겠습니까? [ Y / N ] -> Y 입력 -> 비밀번호 입력 -> 비밀번호가 맞다면 삭제

    > 3. [게임 메인 페이지]
        > 3-1 [캐릭터 정보]
            > 3-1-1. 유저 1의 캐릭터 1이 장비를 착용합니다. -> 장비 착용 완료
            > 3-1-2. 유저 1이 캐릭터 1의 캐릭터 정보를 출력합니다. -> 캐릭터 정보 출력 완료(스탯과 장비, 스킬, 아이템 정보 등)
            > 3-1-3. 캐릭터 1이 레벨 업을 했습니다. -> 스탯 n 포인트 증가 선택, 스킬 레벨업 선택 -> 스탯을 올리시겠습니까? [ Y / N ], 스킬 레벨을 올리시겠습니까? [ Y / N ] -> Y 선택 -> 캐릭터 정보 최신화
            > 3-1-4. 캐릭터 1이 장비 해제를 합니다. -> 장비 해제 -> 캐릭터 정보 최신화 -> 내 아이템 테이블에 착용하고 있던 장비 추가

        > 3-2 [사냥터]
            > 3-2-1. 캐릭터 1이 사냥터 목록을 봅니다. -> 사냥터 선택 후 입장
            > 3-2-2. 사냥 시작 -> n% 확률로 보스 몬스터 등장 -> 보스 몬스터의 외형이 출력 -> 기본 공격 or 스킬 선택해서 보스 몬스터와 대결 -> 데미지를 받거나 줄 때마다 캐릭터 1의 체력 바와 보스몬스터의 체력 바 갱신 -> 보스 퇴치 후 아이템 획득, 메소 획득, 경험치 획득 후 사냥 종료 -> 만약 내가 받은 퀘스트가 있다면 퀘스트 정보 갱신 후 종료
            > 3-2-3. 사냥 시작 -> (100-n)% 확률로 보스 몬스터 등장 X -> 아이템 획득, 메소 획득, 경험치 획득 후 사냥 종료 -> 만약 내가 받은 퀘스트가 있다면 퀘스트 정보 갱신 후 종료

        > 3-3 [경매장]
            > 3-3-1. 경매장에 등록된 아이템을 출력합니다. -> 캐릭터 1이 아이템 선택 후 구매 -> 구매하시겠습니까? [ Y / N ] -> Y 입력 -> 아이템 구매 성공 -> 메소 정보 갱신 -> 원래 없던 아이템이라면 내 아이템 테이블에 등록, 있던 아이템이라면 아이템 테이블에 갯수 업데이트
            > 3-3-2. 캐릭터 1이 경매장에 아이템을 등록합니다 -> 아이템 선택 후 등록 -> 등록하시겠습니까? [ Y / N ] -> Y 입력 -> 아이템 등록 성공 -> 갯수가 1인 아이템이나 2 이상인 아이템을 다 등록했다면 내 아이템 테이블에서 삭제, 갯수가 2 이상인 아이템을 갯수가 남게 등록했다면 아이템 갯수 업데이트

        > 3-4 [상점]
            > 3-4-1. 캐릭터 1이 상점에 입장했습니다. -> 상점 아이템 목록 출력
            > 3-4-2. 캐릭터 1이 상점에서 아이템 1을 구매합니다. -> 캐릭터 1의 아이템 테이블에 아이템 목록 추가 or 갱신, 메소 정보 갱신
            > 3-4-3. 캐릭터 1이 상점에서 아이템 3을 판매합니다. -> 캐릭터 1의 아이템 테이블 목록 삭제 or 갱신, 메소 정보 갱신

        > 3-5 [퀘스트]
            > 3-5-1. 캐릭터 1이 받지 않은 퀘스트 출력 -> 퀘스트 선택 -> 퀘스트를 수락하시겠습니까? [ Y / N ] -> Y 입력 -> 내 캐릭터 퀘스트 테이블에 신규 퀘스트 등록
            > 3-5-2. 캐릭터 1이 받은 퀘스트 완수 -> 퀘스트 선택 -> 퀘스트를 완료하고 보상을 받으시겠습니까? [ Y / N ] -> Y 입력 -> 내 캐릭터 아이템 테이블 정보 갱신 or 메소 정보 갱신

    > 4. [게임 종료]
       > 4-1. 게임 종료 버튼 입력 or 게임 종료 버튼 기능의 숫자 입력 시 [게임 시작 페이지]로 돌아감.
       > 4-2. 로그아웃 버튼 입력 or 로그아웃 버튼 기능의 숫자 입력 시 프로그램 종료.

[시나리오 구현] (GM)
    > 1. [홈페이지]
        > 1-1. 전체 회원 목록을 출력합니다.

    > 2. [게임 시작 페이지]
        > 2-1. 신규 직업 생성이 가능합니다. -> 신규 직업을 생성
        > 2-2. 전체 캐릭터 목록을 출력합니다.

    > 3. [게임 메인 페이지]
        > 3-1. [캐릭터 정보]
            > 3-1-1. 신규 장비 등록, 신규 스킬 추가
            > 3-1-2. 전체 아이템 목록을 출력합니다. 전체 스킬 목록을 출력합니다.
            > 3-1-3. 장비 능력치 조정, 스킬 데미지 조정
            > 3-1-4. 기존 장비 삭제, 기존 스킬 삭제

        > 3-2. [사냥터]
            > 3-1-1. 신규 사냥터 등록, 신규 보스 몬스터 등록
            > 3-1-2. 전체 사냥터 목록 출력, 전체 보스 몬스터 목록 출력
            > 3-1-3. 사냥터 정보(경험치, 아이템, 메소 획득량) 조정, 보스 몬스터 스펙 조정
            > 3-1-4. 사냥터 삭제, 보스 몬스터 삭제

        > 3-3. [경매장]
            > 3-3-1. 전체 경매장 등록 아이템 목록 출력

        > 3-4. [상점]
            > 3-4-1. 상점 전체 아이템 출력
            > 3-4-2. 상점에서 필요한 목록 추가
            > 3-4-3. 불필요한 목록 제거 or 아이템 밸런스 조절

        > 3-5 [퀘스트]
            > 3-5-1. 퀘스트 전체 목록 출력
            > 3-5-2. 신규 퀘스트 출시
            > 3-5-3. 유저들이 많이 하지 않는 퀘스트 삭제 or 조건 하락 or 보상 상향

    > 4. [게임 종료]
        > 4-1. 게임 종료 버튼 입력 or 게임 종료 버튼 기능의 숫자 입력 시 [게임 시작 페이지]로 돌아감.
        > 4-2. 로그아웃 버튼 입력 or 로그아웃 버튼 기능의 숫자 입력 시 프로그램 종료.
